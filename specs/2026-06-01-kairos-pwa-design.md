# Kairos PWA — Design Spec
**Date:** 2026-06-01

## Goal

Make the dashboard installable as a home-screen app named **Kairos** on Android and iOS. Launches fullscreen like a native app, always shows fresh data (no caching). No push notifications in this scope.

## Naming Note

The app's user-facing name is **Kairos** (Greek: "the opportune moment to act"). The underlying repository and pipeline keep their existing `fbtc-timing` names — only the PWA presentation layer and page title adopt the Kairos brand. The app is asset-agnostic by design (Bitcoin today, more assets later).

## What Changes

### 1. `docs/manifest.json` (new file)

Declares the installable app. Because GitHub Pages serves from `https://meyogui.github.io/fbtc-timing/`, **all paths use the `/fbtc-timing/` prefix** — this is critical; a wrong scope silently disables the install prompt.

```json
{
  "name": "Kairos",
  "short_name": "Kairos",
  "description": "Know the opportune moment to invest.",
  "start_url": "/fbtc-timing/",
  "scope": "/fbtc-timing/",
  "display": "standalone",
  "background_color": "#0d0d0d",
  "theme_color": "#0d0d0d",
  "orientation": "portrait",
  "icons": [
    { "src": "/fbtc-timing/icons/icon-192.png", "sizes": "192x192", "type": "image/png", "purpose": "any" },
    { "src": "/fbtc-timing/icons/icon-512.png", "sizes": "512x512", "type": "image/png", "purpose": "any" },
    { "src": "/fbtc-timing/icons/icon-512-maskable.png", "sizes": "512x512", "type": "image/png", "purpose": "maskable" }
  ]
}
```

The maskable icon (purpose `maskable`) lets Android crop the icon into its adaptive shape without clipping content — it has extra padding around the artwork.

### 2. `docs/icons/` (new directory, 3 PNG files)

- `icon-192.png` — 192×192, the icon as designed (artwork fills most of the frame)
- `icon-512.png` — 512×512, same design scaled up
- `icon-512-maskable.png` — 512×512 with the artwork scaled to ~70% inside a safe zone, `#0d0d0d` padding around it, so Android's adaptive masking doesn't clip the waveform or text

Generated once by a helper script (`scripts/generate_icons.py`) that rasterizes the locked SVG via `cairosvg`. Committed as static binary assets — never regenerated by the daily pipeline. No `.gitignore` change is needed: the existing ignore patterns are specific (`replay-*.png`, `dashboard-*.png`), so `docs/icons/*.png` is not ignored and commits normally.

### 3. `docs/sw.js` (new file)

A no-op service worker. Required by Chrome/Android to enable "Add to Home Screen", but deliberately empty of caching logic so every request hits the network and data stays fresh.

```js
// Kairos service worker — intentionally no caching.
// Exists only to satisfy the PWA installability requirement.
// Every request passes through to the network so data is always fresh.
self.addEventListener('install', () => self.skipWaiting());
self.addEventListener('activate', (e) => e.waitUntil(self.clients.claim()));
// No 'fetch' handler — the browser uses the network directly.
```

### 4. `templates/dashboard.html.j2` (modified)

Add to `<head>` (after the existing `<title>`):

```html
  <link rel="manifest" href="/fbtc-timing/manifest.json">
  <meta name="theme-color" content="#0d0d0d">
  <link rel="apple-touch-icon" href="/fbtc-timing/icons/icon-192.png">
  <meta name="apple-mobile-web-app-capable" content="yes">
  <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
  <meta name="apple-mobile-web-app-title" content="Kairos">
```

Change the `<title>` from `FBTC Market Timing` to `Kairos`.

Add before the closing `</body>` tag:

```html
  <script>
    if ('serviceWorker' in navigator) {
      window.addEventListener('load', () => {
        navigator.serviceWorker.register('/fbtc-timing/sw.js').catch(() => {});
      });
    }
  </script>
```

The `apple-mobile-web-app-*` meta tags give iOS the standalone fullscreen behaviour and the "Kairos" name when the user does Share → Add to Home Screen.

## What Does Not Change

- `scripts/score.py`, `scripts/compute_signals.py`, `scripts/build_dashboard.py`, `scripts/fetch_data.py` — untouched
- `.github/workflows/daily.yml` — untouched; it already writes `docs/index.html`, and the new static files (`manifest.json`, `sw.js`, `icons/`) are committed once and served as-is
- No caching, no offline support, no push notifications
- `requirements.txt` — `cairosvg` is only needed locally to generate icons once; it is **not** added to `requirements.txt` (the daily workflow never regenerates icons)

## Verification

1. Run `python scripts/generate_icons.py` → 3 PNGs appear in `docs/icons/`
2. Run `python scripts/build_dashboard.py` → `docs/index.html` contains manifest link and SW registration
3. Push; wait for GitHub Pages deploy
4. On Android Chrome: visit the site → "Add to Home Screen" prompt appears → installs as "Kairos" → opens fullscreen with fresh data
5. On iOS Safari 16.4+: Share → Add to Home Screen → shows "Kairos" + icon → opens fullscreen
6. Confirm the dashboard always shows current data (no stale cache) by checking the "Updated" date matches the latest pipeline run

## Risks / Known Gotchas

- **GitHub Pages path prefix** — the `/fbtc-timing/` prefix in `manifest.json`, `apple-touch-icon`, and the SW registration path must all be correct, or the install prompt silently fails. This is the single most common PWA-on-GitHub-Pages mistake.
- **Icon caching by the OS** — once installed, Android/iOS cache the app icon. Changing the icon later may require reinstalling the PWA to see the update. Not a concern for first install.
